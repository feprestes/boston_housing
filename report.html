Implementação: Definir uma Métrica de Desempenho¶

difcil medir a qualidade de um modelo dado sem quantificar seu desempenho durante o treinamento e teste. Isso geralmente feito utilizando algum tipo de mtrica de desempenho, atravs do clculo de algum tipo de erro, qualidade de ajuste, ou qualquer outra medida til. Para este projeto, voc ir calcular o coeficiente de determinao, R2, para quantificar o desempenho do seu modelo. O coeficiente da determinao para um modelo uma estatstica til em anlise regressa, como se ele frequentemente descrevesse como "good" a capacidade do modelo de fazer estimativas.

Os valores para R2 tm um alcance de 0 a 1, que captura a porcentagem da correlao ao quadrado entre a estimativa e o valor atual da varivel alvo. Um modelo R2 de valor 0 sempre falha ao estimar a varivel alvo, enquanto que um modelo R2 de valor 1, estima perfeitamente a varivel alvo. Qualquer valor entre 0 e 1 indica qual a porcentagem da varivel alvo, ao utilizar esse modelo, ele pode ser explicado pelos atributos. Um modelo pode dar tambm um R2 negativo, que indica que o modelo no melhor do que aquele que estima ingenuamente a mdia da varivel alvo.

Para a funo performance_metric na clula de cdigo abaixo, voc ir precisar implementar o seguinte:

  • Utilizar o r2_score do sklearn.metrics para executar um clculo de desempenho entre y_true e y_predict.
  • Atribuir a pontuao do desempenho para a varivel score.

from sklearn.metrics import r2_score

def performance_metric(y_true, y_predict):
    """ Calcular e retornar a pontuao de desempenho entre 
        valores reais e estimados baseado na mtrica escolhida. """
    
    score = r2_score(y_true, y_predict)
    
    # Devolver a pontuao
    return score

Implementação: Ajustar um Modelo¶

Na sua ltima implementao, voc vai precisar unir tudo o que foi aprendido e treinar um modelo utilizando o algoritmo de rvore de deciso. Para garantir que voc est produzindo um modelo otimizado, voc treinar o modelo utilizando busca em matriz para otimizar o parmetro de profundidade mxima ('max_depth') para uma rvore de deciso. Esse parmetro pode ser entendido como o nmero de perguntas que o algoritmo de rvore de deciso pode fazer sobre os dados antes de fazer uma estimativa. rvores de deciso so parte de uma classe de algoritmos chamados algoritmos de aprendizagem supervisionada.

Para a funo fit_model na clula de cdigo abaixo, voc vai precisar implementar o seguinte:

  • Utilize o DecisionTreeRegressor do sklearn.tree para gerar um objeto regressor de rvore de deciso.
    • Atribua esse objeto varivel 'regressor'.
  • Gere um dicionrio para 'max_depth' com os valores de 1 a 10 e atribua isso para a varivel 'params'.
  • Utilize o make_scorer do sklearn.metrics para gerar um objeto de funo de pontuao.
    • Passe a funo performance_metric como um parmetro para esse objeto.
    • Atribua a funo de pontuao varivel 'scoring_fnc'.
  • Utilize o GridSearchCV do sklearn.grid_search para gerar um objeto de busca por matriz.
    • Passe as variveis 'regressor', 'params', 'scoring_fnc' and 'cv_sets' como parmetros para o objeto.
    • Atribua o objeto GridSearchCV para a varivel 'grid'.

from sklearn.metrics import make_scorer
from sklearn.tree import DecisionTreeRegressor
from sklearn.grid_search import GridSearchCV

def fit_model(X, y):
    """ Desempenhar busca em matriz sobre o parmetro the 'max_depth' para uma 
        rvore de deciso de regresso treinada nos dados de entrada [X, y]. """
    
    # Gerar conjuntos de validao-cruzada para o treinamento de dados
    cv_sets = ShuffleSplit(X.shape[0], n_iter = 10, test_size = 0.20, random_state = 0)

    regressor = DecisionTreeRegressor()

    params = {'max_depth':range(1,11)}

    scoring_fnc = make_scorer(performance_metric)

    grid = GridSearchCV(regressor, param_grid = params, scoring = scoring_fnc, cv = cv_sets)

    # Ajustar o objeto de busca em matriz com os dados para calcular o modelo timo
    grid = grid.fit(X, y)

    # Devolver o modelo timo depois de realizar o ajuste dos dados
    return grid.best_estimator_

O modelo no pode ser considerado robusto o suficiente, e provavelmente o acrscimo de mais atributos poderia ser til para a melhora dele. No caso, possvel perceber que uma variao to grande nos preos poderia ser prejudicial se utilizada no mundo real sem levar em conta outros critrios ou o julgamento humano. Porm, para servir como ponto de partida, o modelo parece ser interessante.
